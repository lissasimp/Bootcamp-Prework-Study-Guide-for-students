HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Use the markdownguide for help with abbreviations
  • HTML is the sturdy base of a website
  • HTML elements are case sensitive
  • Elements that are nested inside others are called child elements
  • The body element holds all the child elements and is known as the parent element
  • An element refers to the entire element from opening tag to closing tag
  • a tag refers only to what appears inside the angle brackets
  • HTML strictly handles the structure and content of a webpage

CSS

  • Cascading Style Sheets (CSS)
  • CSS defines the colors, aesthetics, visual layout, and organization
  • We use CSS to define style rules for specific elements of an HTML page and format them using a wide range of style options.
  • Inline styles apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only. They’re useful for quick and permanent changes
  • The most common way to style a webpage is by using an external CSS style sheet.
  • It allows developers to keep all their CSS rules in a separate file, which makes design changes easier.
  • Separation of concerns is based on the idea that each section of code should have its own responsibility.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.
  • A margin property indicates how much space we want around the outside of an element.
  • The padding property adds space around the content inside an element. When you increase padding, you are actually making the entire element bigger!
  • A star * lets you add the same properties to all elements on a page
  • Two selectors can be combined into a single rule using a comma
  • Do Not Repeat Yourself (DRY) is a coding principle that encourages developers to reuse or share pieces of code in order to reduce the number of lines of code that need to be written or maintained.
  • hexideciaml code for colours https://htmlcolorcodes.com/

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Every project on Git is called a repository or repository
  • A branch is a workspace where you can make changes without affecting the main site
  • Markdown md is a way of writing in plain text that can be easily converted to HTML https://www.markdownguide.org/basic-syntax/#unordered-lists
  • Commit changes - Whenever you add, edit or delete a file you are making a commit
  • When you add an issue, it means you are adding a list of things to do
  • Have one item per issue

JavaScript

  • JavaScript (JS) allows users to interact with websites.
  • Browser Application Programming Interfaces (APIs)
  • Variables are containers that store values. You start by declaring a variable with the let keyword, followed by the name you give to the variable
  • JavaScript is case sensitive.
  • Comments are snippets of text that can be added along with code.
  • An operator is a mathematical symbol that produces a result based on two values (or variables).
  • Conditionals are code structures used to test if an expression returns true or not. A very common form of conditionals is the if...else statement.
  • Functions are a way of packaging functionality that you wish to reuse.
  • Real interactivity on a website requires event handlers. These are code structures that listen for activity in the browser, and run code in response.
  • https://developer.mozilla.org/en-US/docs/Learn/Getting_started_with_the_web/JavaScript_basics
  • The word console refers to a test environment that developers use to check out their code.
  • .log(), is a method, which means a set of instructions that can be executed by a computer. The .log() method will output whatever we add inside the parentheses to the console.
  • Control flow is the order in which a computer executes code in a script.
  • truthy and falsy are actual real words in Javascript!
